Anchor Program Quarterly Report — Q1 2026

Author: Devon Ng, Anchor Program Lead  ·  Date: April 11, 2026

Program Overview

The Anchor Program provides drop-in mental health peer support for youth aged 16 to 22. Q1 2026 logged 312 unique visits across 14 weeks of programming at our Whyte Avenue location.

Outcomes

Outcome data has been difficult to collect this quarter. Our intake survey response rate dropped to 34%, down sharply from 61% in Q4 2025. Several youth raised concerns about wait times to speak with a peer support worker, with three separate complaints documented in the suggestion box.

Concerns

Staff burnout is the most pressing issue I can identify. Both full-time workers have indicated they are considering leaving if conditions don't improve. I have flagged this to the Executive Director on three separate occasions since January and have not received a clear response...

Design specifications for handoff

Color tokens

  • --plum-deep #0F0A1F — base background
  • --plum #2D1B4E — Insyte primary brand
  • --plum-mid #1A1230 — elevated surfaces
  • --accent #B4A4E8 — interactive accent (links, focus, CTA)
  • --accent-soft #9F8FCF — secondary accent

Typography

  • Display headings, page titles, big numbers: Cormorant Garamond, weight 400-500
  • Body, UI, buttons, table content: DM Sans, weight 300-500
  • Eyebrow labels: DM Sans, 11px, uppercase, letter-spacing 0.12em

Atmosphere

  • Dotted grid: 1px dots, rgba(255,255,255,0.04), 32px spacing, fixed background
  • Radial glow: 1200×800px ellipse, plum to transparent, 12-15% opacity, blurred 40px, top-center fixed
  • All cards: glassmorphic with backdrop-filter blur(12px), rgba(255,255,255,0.03) bg, 1px white-alpha border

Motion (use Framer Motion in React)

  • Page mount: cards stagger fade-in with 80ms delay each
  • Analysis run: fake 1.2s loading with subtle accent pulse on the active button
  • Result panel: slides in from below with spring physics
  • Hover: cards lift 2px, border brightens, 200ms ease

What's hardcoded

  • src/data/documents.json — all 21 documents from the markdown file
  • src/data/analysisResults.json — pre-written outputs keyed by analysis type and query
  • src/data/projects.json — sidebar list (one active project plus 3 dummy entries)

What's interactive

  • Picker → routes to /org or /case
  • Analysis buttons → reveal input or run with default query
  • Preview buttons → open document modal
  • Citation chips and inline doc mentions → scroll to or open the relevant document
  • Suggested action chips → trigger pre-built scenario
  • Pattern/risk text input → keyword match against 4-5 prebuilt scenarios with friendly fallback
